Class: MyfoxWrapperApiCommon

MyfoxWrapperApiCommon

Parent class that contains the method list to implement on the subclasses, and common methods. The constructor must never be squizzed even by a subclass: parameters validation is done here.

 The strategy adopted to fallback on another wrapper if the method does not exists. One of the following values: ['htmlOnly', 'htmlFirst', 'restFirst', 'restOnly', 'custom'].

 To automate or not the authentication process (between the wrapper and Myfox services).

 The list of site IDs (linked to the same user account) allowed to be used by the wrapper.

callDistant(url, method, queryParams, headers, payload, streamParser, resolve, reject, reAuthenticate)

Call Myfox interface with the action/query. This method must be overridden in the extended api implementation. Do not use this method directly to call Myfox services. Prefer use this.callApi(url, method, queryParams, headers, payload)
Parameters:
Name Type Description
url string The HTTP(s) URL to call. This url will be parsed in subclasses to replace {siteId} by its value
method string The HTTP method in uppercase (GET, POST, PATCH, PUT, DELETE, etc...)
queryParams object An object to serialize into the query string
headers object An object to push into the request headers
payload object An object to serialize as the request payload
streamParser object A stream parser to pipe on the distant response stream
resolve function The callback to use if the call succeeds (with data as unique parameter)
reject function The callback to use if the call fails (with the error as unique parameter)
reAuthenticate function The callback to use if the call received a 403 error, to force a new loop with an authentication process. May be null ! In this case, throw an error.
